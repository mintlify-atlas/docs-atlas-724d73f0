This guide will walk you through installing and setting up both the frontend and backend components of SANL.

## Prerequisites

Before you begin, ensure you have the following installed:

- **Node.js** 20.x or higher
- **Python** 3.8 or higher
- **npm**, **yarn**, or **pnpm** package manager
- **Git** for cloning the repository

## Frontend Installation

The SANL frontend is built with Next.js 16.1.6 and React 19.2.3.

<Steps>
  <Step title="Navigate to the frontend directory">
    ```bash
    cd frontend
    ```
  </Step>

  <Step title="Install dependencies">
    Choose your preferred package manager:

    <CodeGroup>
      ```bash npm
      npm install
      ```

      ```bash yarn
      yarn install
      ```

      ```bash pnpm
      pnpm install
      ```
    </CodeGroup>

    This will install the following key dependencies:
    - Next.js 16.1.6
    - React 19.2.3
    - React DOM 19.2.3
    - Tailwind CSS 4.x
    - TypeScript 5.x
  </Step>

  <Step title="Start the development server">
    <CodeGroup>
      ```bash npm
      npm run dev
      ```

      ```bash yarn
      yarn dev
      ```

      ```bash pnpm
      pnpm dev
      ```
    </CodeGroup>

    The application will be available at [http://localhost:3000](http://localhost:3000).
  </Step>
</Steps>

## Backend Installation

The SANL backend is built with FastAPI and integrates with Google Gemini AI and Supabase.

<Steps>
  <Step title="Navigate to the backend directory">
    ```bash
    cd backend
    ```
  </Step>

  <Step title="Create a Python virtual environment">
    ```bash
    python3 -m venv venv
    ```

    This creates an isolated Python environment for the project dependencies.
  </Step>

  <Step title="Activate the virtual environment">
    ```bash
    source venv/bin/activate
    ```

    <Note>
      On Windows, use `venv\Scripts\activate` instead.
    </Note>
  </Step>

  <Step title="Install Python dependencies">
    ```bash
    pip install -r requirements.txt
    ```

    This will install the following packages:
    - **fastapi** - Modern web framework for building APIs
    - **uvicorn** - ASGI server for running FastAPI
    - **google-generativeai** - Google Gemini AI SDK
    - **supabase** - Supabase Python client
    - **python-dotenv** - Environment variable management
    - **fpdf2** - PDF generation library
  </Step>

  <Step title="Configure environment variables">
    Create a `.env` file in the backend directory based on `.env.example`:

    ```bash
    cp .env.example .env
    ```

    Then configure your environment variables. See the [Environment Configuration](/setup/environment) guide for details.
  </Step>

  <Step title="Start the FastAPI server">
    ```bash
    uvicorn main:app --reload
    ```

    The API will be available at [http://localhost:8000](http://localhost:8000).
    
    <Info>
      The `--reload` flag enables auto-reload during development. Remove it in production.
    </Info>
  </Step>
</Steps>

## Automated Backend Setup

For convenience, you can use the provided setup script to automate the backend installation:

```bash
./setup_backend.sh
```

This script will:
1. Navigate to the backend directory
2. Create a Python virtual environment
3. Activate the virtual environment
4. Install all required dependencies from `requirements.txt`

<Warning>
  Make sure the script has execute permissions. If needed, run:
  ```bash
  chmod +x setup_backend.sh
  ```
</Warning>

## Verification

After installation, verify that both components are running correctly:

### Frontend Verification

1. Open [http://localhost:3000](http://localhost:3000) in your browser
2. You should see the SANL application interface
3. Check the browser console for any errors

### Backend Verification

1. Open [http://localhost:8000/docs](http://localhost:8000/docs) in your browser
2. You should see the FastAPI interactive API documentation
3. Test a simple endpoint to ensure the API is responding

## Troubleshooting

<AccordionGroup>
  <Accordion title="Node.js version issues">
    Ensure you're using Node.js 20.x or higher. Check your version with:
    ```bash
    node --version
    ```
    
    Use [nvm](https://github.com/nvm-sh/nvm) to manage multiple Node.js versions if needed.
  </Accordion>

  <Accordion title="Python virtual environment issues">
    If you encounter issues with the virtual environment, ensure Python 3.8+ is installed:
    ```bash
    python3 --version
    ```
    
    Try deleting and recreating the virtual environment if problems persist.
  </Accordion>

  <Accordion title="Port already in use">
    If port 3000 or 8000 is already in use, you can specify a different port:
    
    **Frontend:**
    ```bash
    PORT=3001 npm run dev
    ```
    
    **Backend:**
    ```bash
    uvicorn main:app --port 8001 --reload
    ```
  </Accordion>

  <Accordion title="Dependency installation failures">
    If you encounter issues installing dependencies:
    
    1. Clear your package manager cache
    2. Delete `node_modules` (frontend) or `venv` (backend)
    3. Reinstall dependencies from scratch
    4. Ensure you have a stable internet connection
  </Accordion>
</AccordionGroup>
