## Google Gemini 1.5 Flash

### Why Gemini 1.5 Flash?

Gemini 1.5 Flash is selected for SANL based on:

- **Speed:** Fast inference for real-time nutritional analysis
- **Cost-effectiveness:** Optimized pricing for production deployment
- **Quality:** High-quality responses for medical and nutritional contexts
- **Context window:** Large context for processing comprehensive patient data
- **Multimodal:** Support for text and future image-based dietary analysis

## Integration with FastAPI

Create an endpoint for nutritional analysis:

```python
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel
import google.generativeai as genai

app = FastAPI()

# Configure Gemini
genai.configure(api_key=os.getenv("GEMINI_API_KEY"))
model = genai.GenerativeModel('gemini-1.5-flash')

class NutritionalAssessment(BaseModel):
    patient_id: str
    height_cm: float
    weight_kg: float
    age: int
    dietary_intake: dict
    medical_conditions: list[str] = []

@app.post("/api/analyze-nutrition")
async def analyze_nutrition(assessment: NutritionalAssessment):
    try:
        # Calculate BMI
        bmi = assessment.weight_kg / ((assessment.height_cm / 100) ** 2)
        
        # Create prompt
        prompt = f"""
        Analyze the following nutritional data:
        
        Patient: {assessment.patient_id}
        Age: {assessment.age}
        Height: {assessment.height_cm}cm
        Weight: {assessment.weight_kg}kg
        BMI: {bmi:.2f}
        Medical Conditions: {', '.join(assessment.medical_conditions)}
        
        Dietary Intake:
        {assessment.dietary_intake}
        
        Provide a comprehensive nutritional analysis with recommendations.
        """
        
        # Generate analysis
        response = model.generate_content(prompt)
        
        return {
            "patient_id": assessment.patient_id,
            "bmi": round(bmi, 2),
            "analysis": response.text,
            "model_version": "gemini-1.5-flash"
        }
        
    except Exception as e:
        raise HTTPException(status_code=500, detail=f"AI analysis failed: {str(e)}")
```

## Response Handling

### Parsing AI Responses

```python
import json
import re

def parse_gemini_response(response_text: str) -> dict:
    """
    Parse Gemini response into structured data
    """
    try:
        # Try to extract JSON if present
        json_match = re.search(r'```json\n(.+?)\n```', response_text, re.DOTALL)
        if json_match:
            return json.loads(json_match.group(1))
        
        # Otherwise return as text with metadata
        return {
            "analysis": response_text,
            "format": "text"
        }
    except Exception as e:
        return {
            "analysis": response_text,
            "format": "text",
            "parse_error": str(e)
        }
```

### Implementing Rate Limiting

```python
from time import time
from collections import deque

class RateLimiter:
    def __init__(self, max_requests=15, time_window=60):
        self.max_requests = max_requests
        self.time_window = time_window
        self.requests = deque()
    
    def is_allowed(self) -> bool:
        now = time()
        
        # Remove old requests outside time window
        while self.requests and self.requests[0] < now - self.time_window:
            self.requests.popleft()
        
        # Check if under limit
        if len(self.requests) < self.max_requests:
            self.requests.append(now)
            return True
        
        return False

# Usage
rate_limiter = RateLimiter()

if not rate_limiter.is_allowed():
    raise HTTPException(status_code=429, detail="Rate limit exceeded. Please try again later.")
```

### Anonymizing Patient Data

```python
import hashlib

def anonymize_patient_data(patient_data: dict) -> dict:
    """
    Remove PII before sending to AI
    """
    # Hash patient ID
    patient_id_hash = hashlib.sha256(
        patient_data["patient_id"].encode()
    ).hexdigest()[:16]
    
    return {
        "patient_id": patient_id_hash,
        "age": patient_data["age"],
        "height_cm": patient_data["height_cm"],
        "weight_kg": patient_data["weight_kg"],
        "medical_conditions": patient_data["medical_conditions"],
        "dietary_intake": patient_data["dietary_intake"]
        # Exclude: name, DOB, contact info, medical record number
    }
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="API key authentication failures">
    **Symptoms:** 401 errors or "API key not valid"
    
    **Solutions:**
    1. Verify `GEMINI_API_KEY` is set correctly in `.env`
    2. Check the key hasn't been revoked in Google AI Studio
    3. Ensure no extra whitespace in the key
    4. Restart the backend server after updating the key
  </Accordion>

  <Accordion title="Rate limit exceeded errors">
    **Symptoms:** 429 errors or "Resource exhausted"
    
    **Solutions:**
    1. Implement rate limiting in your application
    2. Cache common AI responses
    3. Batch similar requests together
    4. Consider upgrading to a paid plan
    5. Use exponential backoff for retries
  </Accordion>

  <Accordion title="Poor quality AI responses">
    **Symptoms:** Irrelevant or inaccurate nutritional advice
    
    **Solutions:**
    1. Improve prompt engineering with more context
    2. Add examples of desired output format
    3. Set temperature parameter lower for more deterministic responses
    4. Include relevant medical guidelines in the prompt
    5. Implement human review workflow
  </Accordion>

  <Accordion title="Slow response times">
    **Symptoms:** API calls taking too long
    
    **Solutions:**
    1. Use async/await for non-blocking calls
    2. Implement timeout limits
    3. Reduce prompt size and complexity
    4. Cache frequent analyses
    5. Consider streaming responses for large outputs
  </Accordion>
</AccordionGroup>
